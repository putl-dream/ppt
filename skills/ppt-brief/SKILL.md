# PPT Brief 起草

## 目标

Task 子 Agent 创建精简的 `brief.md`。信息已足则直接写；仅缺关键项时 AskUser **一次**（最多 1–3 问）。

## 需求澄清（内容阶段 · 不涉及视觉）

用户只给主题或模糊想法时，优先对齐以下项（已明确则跳过）：

| # | 字段 | 为什么要问 |
|---|------|------------|
| 1 | **受众与场景** | 内部分享 / 竞聘 / 发布 / demo |
| 2 | **分享时长** | 15min≈10页 · 30min≈20页 · 45min≈25–30页 |
| 3 | **原始素材** | 文档/数据/旧稿/链接 |
| 4 | **图片计划** | 有无截图；后期占位即可 |
| 5 | **硬约束** | 必含数据 / 禁出现内容 |

**本阶段不问**：主题色、杂志风/瑞士风、版式节奏——这些在排版阶段（layout-plan）再定。

## brief.md 结构（保持简短）

```markdown
# 演示标题

## 目的
（一句话）

## 受众
（一句话）

## 时长与页数
（例如 30 分钟 · 约 20 页）

## 素材
（有/无 · 简述）

## 要点
- 要点 1
- 要点 2
```

## 工作流

1. 用户已给完整大纲 → 可跳过 brief，直接 outline/storyboard。
2. 缺 1–2 项 → AskUser 一次；不要连环追问。
3. 完成后主 Agent 摘要 2–3 句，不粘贴全文。

## 质量

- 目的与受众各一句话。
- 页数与时长合理即可。
- 文案可完整表达，**不必在 brief 阶段压缩字数**。

## 衔接

完成后 LoadSkill `ppt-outline`。
