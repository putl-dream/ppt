# 商业视觉质量评分与人工复核规范

> 版本：commercial-visual-human-v1  
> 日期：2026-07-21  
> 适用范围：Lean Commercial Visual Compiler v2

## 1. 评分边界

机器质量分只表示可计算的结构和视觉信号，不代表商业审美结论。报告必须同时包含：

- `scores`：机器评分；
- `scoreDetails`：每项分数的状态和证据；
- `humanReview.status`：人工复核状态；
- `brandSignals`：token 一致性、首尾母题连续性，以及尚未机器评分的视觉辨识度。

当整套 Deck 明确声明 `imageMode=none` 时，`assetQuality` 为 `null`，状态为 `not-applicable`，不得用强视觉 Scene 数量替代素材质量并给出 100 分。若页面声明图片意图但导演没有生成素材请求，则素材质量为 0，并产生 `asset-intent-unplanned` warning。

## 2. 初始自动基准

| 基准 | 输入 | 预期结果 |
|---|---|---|
| 原生无图商业稿 | `commercial-visual-growth-os.json` | 素材维度不适用；机器分低于 90；人工状态为未复核 |
| 图片意图被回避 | 将页面 `imageMode` 设为 `optional/required`，但不生成 asset request | 素材分为 0；报告包含 `asset-intent-unplanned` |
| 重复卡片墙 | 25% 以上页面使用三个或更多大卡片，且没有图片、图表或主指标 | 报告包含 `card-wall-overuse`；`cardWallControl` 明显扣分 |

这些基准分别由 `commercial-ppt-fixture.test.ts` 和 `commercial-visual-quality.test.ts` 固化。测试期望是需求契约，不得为迁就实现而放松。

## 3. 人工评分表

每份基准稿由至少三名评审按 1–5 分独立评分：

| 维度 | 1 分 | 3 分 | 5 分 |
|---|---|---|---|
| 视觉辨识度 | 接近通用模板，缩略图无法识别人格 | 有稳定风格但母题较弱 | 缩略图即可识别，且跨页一致 |
| 信息冲击力 | 页面没有明确第一视线 | 结论可见但竞争元素较多 | 3 秒内能识别结论与证据 |
| 品牌适配 | 与受众、行业或场景冲突 | 基本符合但缺少专属性 | 气质、字体、色彩和素材共同服务品牌 |

同时记录：

- 是否需要结构重做；
- 预计人工修改分钟数；
- 是否可以直接进入客户内部评审；
- 最弱页面及原因。

机器分与人工分必须分别展示。机器排序只在累计至少 15 套真实基准稿和三人评分后计算相关性，不得在此前宣称机器分等同于审美分。

## 4. 阶段验收

质量门改动至少满足：

1. 每个机器维度提供可复现证据；
2. 不适用维度不进入总分平均值；
3. Scene 名称本身不能证明页面存在视觉焦点；
4. 重复大卡片结构会被识别并扣分；
5. 视觉辨识度、信息冲击力和品牌适配必须保持 `not-reviewed`，直至真实人工评分写入报告。
